History

MapOSMatic has been started through the idea of Gilles Lamiral, an OpenStreetMap and free software contributor from the area of Rennes, France. Then, a group of crazy hackers, grouped together during a one-week Hackfest in August 2009 brought the idea of Gilles Lamiral to life by writing the code and named the project MapOSMatic. Through this page, the group of crazy hackers would like to thank Gilles for sharing his bright idea.

Of course, MapOSMatic is fully free software, licensed under the AGPLv3.

How does it work ?

On our server, we run a PostgreSQL server, with the PostGIS extension. In this PostgreSQL server, we have loaded the full OpenStreetMap data for the metropolitan France, using the dumps and the osm2pgsql tool.

For the map rendering, we use the famous Mapnik with the OpenStreetMap stylesheet available in OpenStreetMap Subversion repository. Using Mapnik, we built OCitySMap, a Python module that :

  • Renders the map using OpenStreetMap data and the OpenStreetMap Mapnik stylesheet
  • Renders on top of the map, using Mapnik, the grid of 500 meters squares, the labels of the squares, and a few other informations on the map (copyright, scale, etc.)
  • Renders an index of the streets, based on OpenStreetMap data and the intersection of the streets with the squares on the map

This Python module can be used through a command-line provided with OCitySMap, so everyone can run its own city-map rendering suite. However, as the installation of the different components is quite complicated, a small web service has been created on top of it.

MapOSMatic is the association of a small web service developed on top of the Django framework and a daemon that processes the rendering requests.
